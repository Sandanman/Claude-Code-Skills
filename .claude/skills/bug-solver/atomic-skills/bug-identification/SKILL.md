# Bug Identification 原子Skill

## 概述
收集和整理用户提供的bug信息，创建结构化的问题描述文档，为后续分析提供基础。

## 核心能力
- 提取用户描述中的关键信息
- 识别错误类型（前端/后端/配置）
- 整理复现步骤
- 收集环境信息
- 输出标准化的问题报告

## 输入
用户对bug的描述（文本）

## 输出
结构化的bug报告文档，包含以下字段：

```markdown
# Bug 问题识别报告

## 基础信息
- 问题类型：[前端/后端/配置/其他]
- 严重程度：[高/中/低]
- 发生环境：[浏览器/操作系统/设备/网络环境]
- 发生频率：[偶发/稳定复现/随机]

## 问题描述
用户原始描述：
[原始描述]

## 复现步骤
1. [步骤1]
2. [步骤2]
3. [步骤3]
...

## 期望行为
[预期的正确行为]

## 实际行为
[实际观察到的行为]

## 相关信息
- 错误日志：[控制台错误、服务端错误等]
- 截图/录屏：[如有]
- 相关代码文件：[可能涉及的文件路径]
- 环境信息：[浏览器版本、Node版本、依赖版本等]

## 问题分类
- 所属模块：[登录/会议/设置/其他]
- 问题性质：[UI问题/逻辑错误/性能问题/兼容性问题/配置错误]

## 初步判断
[基于初步信息的猜测，如：可能与API调用失败有关]
```

## 执行逻辑
1. 分析用户输入，提取关键信息
2. 识别错误类型和所属模块
3. 根据用户描述整理复现步骤
4. 询问并补充缺失的关键信息（如错误日志、环境信息）
5. 输出结构化报告

## 依赖关系
- 无依赖

## 完成标准
1. 问题类型已明确分类
2. 复现步骤完整清晰
3. 期望行为与实际行为对比明确
4. 至少包含错误日志或相关代码文件信息
5. 输出完整结构化报告

## 错误处理
- 如果用户描述模糊，询问关键信息：
  - "能提供完整的错误日志吗？"
  - "具体的复现步骤是什么？"
  - "在哪种环境下会出现这个问题？"
- 如果信息不足，暂停等待用户补充，不继续后续步骤

## 示例

**输入：**
"登录页面点击登录按钮没有反应，控制台显示TypeError: Cannot read property 'login' of undefined"

**输出：**
```markdown
# Bug 问题识别报告

## 基础信息
- 问题类型：前端
- 严重程度：高
- 发生环境：浏览器
- 发生频率：稳定复现

## 问题描述
用户原始描述：
登录页面点击登录按钮没有反应，控制台显示TypeError: Cannot read property 'login' of undefined

## 复现步骤
1. 打开登录页面
2. 输入用户名和密码
3. 点击登录按钮

## 期望行为
点击登录按钮后，应调用登录API，验证成功后跳转到首页

## 实际行为
点击登录按钮后，无任何反应，控制台显示TypeError: Cannot read property 'login' of undefined

## 相关信息
- 错误日志：TypeError: Cannot read property 'login' of undefined
- 相关代码文件：src/views/Login/index.vue, src/api/user.js
- 环境信息：Chrome 123, Node.js 18.17.0

## 问题分类
- 所属模块：登录
- 问题性质：逻辑错误

## 初步判断
可能与登录API调用参数或对象属性访问有关，可能userAPI对象未正确初始化
```

## 注意事项
- 保持客观，不猜测未证实的细节
- 优先记录事实，再进行判断
- 信息不足时，必须询问补充，不能跳过
