# Fix Verification 原子Skill

## 概述
验证修复方案是否真正解决了问题，并检查是否引入了新的问题。

## 核心能力
- 验证修复是否解决问题
- 检查是否引入新问题
- 对比修复前后差异
- 输出验证报告

## 输入
fix-generation输出的修复方案文档

## 输出
修复验证报告，包含以下内容：

```markdown
# 修复验证报告

## 验证目标
[明确验证的目标，如：验证修复是否解决了TypeError: Cannot read property 'login' of undefined]

## 验证方法
[详细描述验证步骤]

### 方法1：[主要验证方法]
- **步骤**：
  1. [步骤1]
  2. [步骤2]
  3. [步骤3]
- **预期结果**：[预期的正确行为]
- **实际结果**：[实际观察到的行为]
- **结论**：[成功/失败]

### 方法2：[辅助验证方法]
- **步骤**：
  1. [步骤1]
  2. [步骤2]
  3. [步骤3]
- **预期结果**：[预期的正确行为]
- **实际结果**：[实际观察到的行为]
- **结论**：[成功/失败]

## 验证结果
| 验证点 | 方法 | 结果 | 说明 |
|--------|------|------|------|
| 问题是否解决 | 方法1 | [是/否] | [详细说明] |
| 是否引入新问题 | 方法1 | [是/否] | [详细说明] |
| 功能完整性 | 方法2 | [是/否] | [详细说明] |
| 性能影响 | 方法2 | [是/否] | [详细说明] |

## 修复前后对比
### 修复前
- **错误表现**：[描述修复前的问题]
- **错误日志**：[错误日志内容]
- **复现步骤**：[复现步骤]

### 修复后
- **错误表现**：[描述修复后的状态]
- **错误日志**：[无错误日志]
- **复现步骤**：[复现步骤]

## 问题确认
- [ ] 是否稳定复现？
- [ ] 是否在所有环境都正常？
- [ ] 是否有其他功能受影响？
- [ ] 是否需要额外测试？

## 验证结论
[明确的结论：修复成功/修复失败/部分成功]

## 失败分析
[如果验证失败，详细分析原因]

## 建议
[如果验证失败，提供改进建议]

## 验证报告摘要
[简洁的总结，用于快速阅读]
```

## 执行逻辑
1. 根据修复方案中的验证建议进行验证
2. 执行至少两个验证方法（主要+辅助）
3. 记录修复前后对比
4. 评估是否引入新问题
5. 输出完整的验证报告

## 依赖关系
- 依赖：fix-generation（必须有完整的修复方案文档）

## 完成标准
1. 至少执行两个验证方法
2. 包含修复前后对比
3. 明确结论：修复成功/失败/部分成功
4. 评估是否引入新问题
5. 输出完整的验证报告

## 错误处理
- 如果验证失败，记录具体失败原因
- 如果验证环境不一致，标记为"需要在特定环境验证"
- 如果无法复现问题，标记为"无法复现"

## 示例

**输入：**
fix-generation输出的修复方案中包含：
- 推荐方案：修正导入路径，将`import userAPI from '@/utils/utils'`改为`import userAPI from '@/api/user.js'`
- 修复验证建议：重启开发服务器，清除浏览器缓存，重新打开登录页面，输入有效用户名和密码，点击登录按钮，验证是否正常跳转到首页，检查控制台无任何错误

**输出：**
```markdown
# 修复验证报告

## 验证目标
验证修复是否解决了登录功能中"TypeError: Cannot read property 'login' of undefined"的错误

## 验证方法

### 方法1：功能验证（主要方法）
- **步骤**：
  1. 重启开发服务器
  2. 清除浏览器缓存
  3. 打开登录页面
  4. 输入用户名：testuser，密码：password123
  5. 点击登录按钮
- **预期结果**：登录成功，跳转到首页，控制台无任何错误
- **实际结果**：登录成功，跳转到首页，控制台无任何错误
- **结论**：成功

### 方法2：代码审查（辅助方法）
- **步骤**：
  1. 检查src/views/Login/index.vue文件的修改
  2. 验证导入路径是否正确：`import userAPI from '@/api/user.js'`
  3. 验证userAPI模块是否存在且导出正确
- **预期结果**：导入路径正确，userAPI模块存在且导出login方法
- **实际结果**：导入路径正确，userAPI模块存在且导出login方法
- **结论**：成功

## 验证结果
| 验证点 | 方法 | 结果 | 说明 |
|--------|------|------|------|
| 问题是否解决 | 方法1 | 是 | 登录成功，跳转到首页，无错误 |
| 是否引入新问题 | 方法1 | 否 | 无其他功能异常 |
| 功能完整性 | 方法2 | 是 | 导入路径正确，模块正常 |
| 性能影响 | 方法2 | 否 | 无性能下降 |

## 修复前后对比
### 修复前
- **错误表现**：点击登录按钮无反应，控制台显示TypeError: Cannot read property 'login' of undefined
- **错误日志**：TypeError: Cannot read property 'login' of undefined
- **复现步骤**：打开登录页面，输入用户名和密码，点击登录按钮

### 修复后
- **错误表现**：点击登录按钮正常跳转到首页，无任何错误
- **错误日志**：无错误日志
- **复现步骤**：打开登录页面，输入用户名和密码，点击登录按钮

## 问题确认
- [x] 是否稳定复现？
- [x] 是否在所有环境都正常？
- [x] 是否有其他功能受影响？
- [x] 是否需要额外测试？

## 验证结论
修复成功：登录功能已恢复正常，问题已解决，未引入新问题。

## 失败分析
[无]

## 建议
[无]

## 验证报告摘要
修复成功：通过功能验证和代码审查，确认登录功能已恢复正常，问题已解决，未引入新问题。
```

## 注意事项
- 验证必须基于真实环境，不能仅凭代码分析
- 优先使用功能验证，代码审查作为辅助
- 所有验证结果必须可复现
- 保持客观，如实记录结果
