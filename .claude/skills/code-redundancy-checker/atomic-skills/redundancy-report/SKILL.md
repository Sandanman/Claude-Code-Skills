# Redundancy Report 原子Skill

## 概述
汇总重复代码检测和死代码检测的结果，生成完整的可操作冗余报告。对每个冗余点进行严重程度分级（高/中/低），按优先级排序，并提供具体的修复建议和预估收益。

## 核心能力
- 汇总两个检测阶段的输出
- 严重程度分级（高/中/低）
- 修复优先级排序
- 冗余统计与指标计算
- 修复收益预估
- 生成 Markdown 格式报告
- 估算清理工作量

## 报告结构

### 1. 执行摘要
- 扫描范围（文件数、代码行数）
- 检测到冗余总数
- 严重程度分布（高/中/低）
- 预估可删除代码行数
- 预估性能收益

### 2. 冗余分类汇总

#### 重复代码汇总
- 文件内重复代码块数
- 跨文件重复代码块数
- 可提取为公共函数的重复代码
- 总体重复率（目标 < 5%）

#### 死代码汇总
- 未使用的 export 数
- 未使用的函数数
- 未使用的变量数
- 未使用的 import 数
- 可删除代码行数估算

### 3. 详细清单（按优先级排序）

#### P0 - 高优先级（立即处理）
- 跨文件完全重复代码（100% 匹配）
- 未使用的 export（浪费打包体积）
- 重复 3 次以上的代码块

#### P1 - 中优先级（计划处理）
- 文件内重复代码
- 高度相似代码（70%~99%）
- 未使用的函数（确认后删除）

#### P2 - 低优先级（有空处理）
- 冗余 import
- 未使用的 props
- 中度相似代码（50%~69%）

### 4. 修复建议

对每个冗余点，提供：
- **位置**：精确文件路径和行号
- **当前问题**：冗余类型和代码片段
- **修复方案**：具体操作步骤
- **预估收益**：删除后可减少的代码行数、预期性能提升
- **风险等级**：低/中/高（修改可能带来的风险）

### 5. 修复行动计划

建议的清理顺序：
1. 删除未使用的 import（零风险）
2. 删除未使用的 export 和函数（需确认无引用）
3. 提取跨文件重复代码到公共模块
4. 消除文件内重复代码

## 输入
- duplicate-code-detection 输出的重复代码清单
- unused-code-detection 输出的死代码清单

## 完成标准
- 报告包含所有冗余点的详细信息
- 每个冗余点有严重程度、位置、修复建议
- 提供清理优先级和行动计划
- 预估清理收益（代码行数、包体积等）
- 支持导出为 Markdown 格式

## 输出格式
- 默认输出 Markdown 格式报告
- 可选输出 JSON 格式（便于程序处理）
- 报告保存路径：`./reports/redundancy-report-{timestamp}.md`

## 注意事项
- 不生成任何修复后的代码，仅提供修复建议
- 对每个冗余点标注风险等级
- 提供多个修复方案选项（简化/完整）
- 区分"可直接删除"和"需确认后删除"两类
