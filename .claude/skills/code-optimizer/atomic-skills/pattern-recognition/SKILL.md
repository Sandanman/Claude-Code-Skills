# Pattern Recognition 原子Skill

## 概述
基于代码质量分析报告，识别具体的可优化模式，定位代码中的反模式和低效实现，为后续生成优化建议提供具体目标。

## 核心能力
- 识别低效算法模式（时间复杂度优化）
- 检测冗余逻辑模式（重复计算、冗余判断）
- 发现不良实践模式（反模式、违反原则）
- 定位性能瓶颈（频繁DOM操作、过度渲染、内存泄漏）
- 提取可复用模式（组件拆分、函数提取）

## 支持的模式类型

### 1. 性能相关模式
- **低效算法**：O(n²) → O(n)，重复计算，不必要的遍历
- **内存泄漏**：未清理的事件监听器、定时器、闭包持有大对象引用
- **渲染性能**：频繁DOM操作、大列表未虚拟化、未使用memo/useMemo缓存

### 2. 可维护性相关模式
- **大组件模式**：单组件超过300行，职责过多
- **重复逻辑模式**：相同条件判断、数据转换逻辑重复
- **过长函数模式**：函数超过50行，包含多个抽象层级

### 3. 代码质量相关模式
- **嵌套条件模式**：三重以上if-else嵌套，可使用卫语句简化
- **参数过多模式**：函数参数超过5个，可使用对象封装
- **硬编码模式**：魔术数字/字符串未定义常量

### 4. 安全相关模式
- **XSS风险模式**：使用v-html或innerHTML渲染用户输入
- **不安全API模式**：使用eval()、document.write()、直接拼接SQL

## 输入
code-quality-analysis输出的质量分析报告

## 执行逻辑
1. 接收code-quality-analysis的质量分析报告
2. 分析报告中的问题列表
3. 对每个问题进行模式匹配
4. 提取具体代码段进行分析
5. 评估优化收益和修复难度
6. 按优先级排序
7. 分析模式间的关联性
8. 输出详细的模式识别报告

## 依赖关系
- 依赖：code-quality-analysis（必须有完整的质量分析报告）
- 被依赖：improvement-suggestion

## 完成标准
1. 至少识别出3种不同的优化模式
2. 每个模式包含位置、问题描述、根本原因
3. 包含模式分类和预期收益
4. 提供优先级排序（高/中/低）
5. 包含模式间关联性分析
6. 输出完整格式化的报告（markdown）

## 错误处理
- **质量报告为空**：报错并退出
- **代码文件读取失败**：跳过该模式并记录警告
- **模式无法归类**：标记为"其他模式"并记录

## 注意事项
- 模式识别基于通用最佳实践，具体项目可能有特殊需求
- 某些"反模式"可能是业务需要的权衡，需人工确认
- 优先级排序综合考虑了影响范围、修复难度和收益
- 关联性分析有助于制定最优的修复顺序
