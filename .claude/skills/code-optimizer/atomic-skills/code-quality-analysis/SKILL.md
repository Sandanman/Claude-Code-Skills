# Code Quality Analysis 原子Skill

## 概述
对目标代码进行静态分析，系统化地评估代码质量，识别问题点，为后续优化提供数据基础。

## 核心能力
- 计算复杂度指标（圈复杂度、认知复杂度、函数长度）
- 检测重复代码段
- 识别常见代码异味
- 检查安全风险点
- 评估代码可维护性

## 分析维度

### 1. 复杂度指标
- **圈复杂度（Cyclomatic Complexity）**：控制流程复杂度，目标值 < 15
- **认知复杂度（Cognitive Complexity）**：人类理解的难度，目标值 < 10
- **函数行数（Lines of Function）**：单函数代码行数，目标值 < 50
- **文件行数（Lines of File）**：文件总行数，建议 < 300

### 2. 代码重复
- **重复代码率**：相同或相似代码块的比例，目标 < 5%
- **克隆检测**：识别代码克隆区域
- **重复逻辑**：相同的条件判断、异常处理出现多次

### 3. 代码异味
- **大类**：单类行数过多或方法过多
- **长参数列表**：函数参数 > 5个
- **重复代码**：多处出现相同代码块
- **长方法**：单个方法 > 50行
- **过多条件**：大量if-else或switch-case嵌套
- **魔术数字/字符串**：未命名的硬编码值
- **全局变量滥用**：过多的全局状态
- **循环嵌套**：嵌套层数 > 2层
- **过深嵌套**：if/循环嵌套深度 > 3层
- **过大表达式**：单个表达式过长（> 40字符）

### 4. 安全风险
- **硬编码密钥**：API密钥、密码出现在代码中
- **eval()使用**：存在eval()调用
- **不安全的API**：使用已知的不安全函数（如innerHTML直接插入）
- **未验证输入**：缺少输入验证

## 输入
- 需要分析的代码文件路径或目录路径
- 可选：额外的分析规则配置文件

## 执行逻辑
1. 接收目标代码文件或目录路径
2. 递归扫描所有相关文件（.js, .ts, .vue, .jsx, .tsx等）
3. 对每个文件执行静态分析，计算各项指标
4. 汇总分析结果，生成详细报告
5. 按严重程度排序问题
6. 识别优化模式（为后续提供输入）

## 依赖关系
- 依赖：无（首个原子skill）
- 被依赖：pattern-recognition, performance-analysis

## 完成标准
1. 至少分析了1个文件（如果是目录则至少3个文件）
2. 计算出平均圈复杂度、认知复杂度、重复代码率
3. 至少识别出3种不同的代码异味
4. 包含安全风险检查
5. 提供优化优先级排序（高/中/低）
6. 输出完整格式化的报告（markdown）

## 错误处理
- **文件不存在**：报错并退出
- **文件为空**：跳过并记录警告
- **语法错误**：跳过文件，记录错误位置
- **缺少依赖**：检查是否安装了必要的工具

## 注意事项
- 这是一个静态分析，不会修改任何代码
- 分析基于通用标准，可能需要根据项目定制
- 优先级排序综合考虑了影响范围和修复难度
- 某些"异味"可能是业务需要，需要人工判断
