## 🚀 如何使用

### 快速开始
1. **阅读顺序**：README.md → SKILL.md
2. **深入了解**：根据需要查阅 algorithms.md/config.md 等
3. **调整配置**：修改 config.md
4. **定制交互**：编辑 user_interaction.md

### 实现时的工作流
```
1. 读取 SKILL.md (核心职责)
   ↓
2. 执行到某个步骤时
   ↓
3. 根据 SKILL.md 中的【执行时按需读取】指令
   ↓
4. 动态加载对应的参考文档 (algorithms.md/config.md 等)
   ↓
5. 继续执行
```

## 📖 文件详细说明

### SKILL.md
- **类型**：核心技能定义文件，标准 YAML frontmatter 格式
- **内容**： orchestrator 的核心职责、9个执行步骤、流程图
- **何时使用**：用户调用 orchestrator skill 时首先读取此文件

### algorithms.md
- **类型**：算法参考文档
- **内容**：8个核心算法（意图识别、相似度计算、依赖分析等）的伪代码
- **何时使用**：实现时需要理解算法逻辑时查阅

### config.md
- **类型**：配置参数文档
- **内容**：所有可调用的阈值、超时、限制等参数
- **何时使用**：需要调整系统行为时

### technical_implementation.md
- **类型**：技术实现参考
- **内容**：底层代码示例（文件锁、原子写入等）
- **何时使用**：需要编写底层实现时

### user_interaction.md
- **类型**：用户交互模板
- **内容**：6个决策点的提示格式、选项、默认行为
- **何时使用**：需要修改用户提示时

### skills_register.md
- **类型**：技能注册表
- **内容**：
  - 注册的主skill（Reasoner读取）
  - 注册的原子skill（对应的skill读取）
- **如何维护**：新增 skill 时在此注册，删除 skill 时移除

### missing_skills.md
- **类型**：缺失技能列表
- **内容**：系统中需要的但尚未实现的技能
- **何时生成**：Reasoner 匹配失败时自动记录
- **如何维护**：实现后从列表中移除

### skill_design_thought.md
- **类型**：设计思想文档
- **内容**：原始设计思路、流程图等
- **状态**：历史文档，维护优先级低
- **建议**：可作为参考，但主要逻辑在 SKILL.md

## 🛠️ 管理建议

### 日常维护
1. **不要删除**：除非你确定不再需要某个功能
2. **优先修改**：config.md（参数调整）> user_interaction.md（提示优化）> SKILL.md（核心逻辑）
3. **新增功能**：
   - 新增算法 → algorithms.md
   - 新增配置 → config.md
   - 新增skill → skills_register.md

### 文件大小管理
如果文件过大（>500行）：
- 考虑拆分
- 保持核心逻辑简洁
- 详见 SKILL.md 已按需拆分算法

### 版本控制
- 所有文件都应该提交到 Git
- 重要变更需同步更新版本号
- 在 SKILL.md 底部标注 `最后更新` 时间

---

## ⚠️ 注意事项

1. **SKILL.md 是入口**：不要修改其基本结构，只调整内容
2. **按需读取机制**：参考文档在运行时才加载，不是一次性全部加载
3. **参数统一**：所有配置参数在 config.md，不要散落在其他文件
4. **技能注册**：新增技能必须更新 skills_register.md

## 🤝 问题排查

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| 找不到某个算法 | algorithms.md 章节编号不对 | 按文件目录结构重新编号 |
| 参数不生效 | config.md 参数名错误 | 检查参数拼写 |
| 提示模板错误 | user_interaction.md 格式不对 | 参考现有模板格式 |
| skill未识别 | skills_register.md 未注册 | 添加到相应列表 |
