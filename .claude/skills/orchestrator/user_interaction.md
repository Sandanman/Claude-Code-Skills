# 用户交互决策点

本文档定义 Orchestrator Reasoner 的所有用户交互点，包括提示格式、选项和默认行为。

---

## 1. 决策点汇总表

| 决策点编号 | 触发位置 | 触发条件 | 选项数量 | 超时（默认行为） |
|-----------|---------|---------|---------|----------------|
| D1 | 步骤2.3.1 | 发现未完成相关任务 | 2 | 5分钟 → 开始新任务 |
| D2 | 步骤3.2 | 多个主skill匹配 | 3 | 30秒 → 选择匹配度最高 |
| D3 | 步骤5.1 | 原子skill失败影响后续 | 4 | 3分钟 → 重试 |
| D4 | 步骤6.3 | 动态调整执行计划 | 2 | 2分钟 → 继续 |
| D5 | 步骤7.5 | 3次反思后仍失败 | 3 | 5分钟 → 输出当前结果 |
| D6 | 步骤9.3 | 检测到可固化文档 | 3 | 1分钟 → 仅索引 |

## 2. 详细提示模板

### D1 - 未完成任务发现

**触发条件**：在 `.skills/tasks/current/` 下找到相关未完成任务

**提示格式**：
```
【任务发现】

检测到相关未完成任务:
- 任务ID: {task_id}
- 用户意图: {intent}
- 已完成: {completed_count}/{total_count}个原子skill
- 创建时间: {create_time}

请选择:
1. 继续执行未完成任务（从断点恢复）
2. 开始新任务（忽略旧任务）

输入 [1/2]: _
```

**处理逻辑**：
- 选择1：继续执行，执行到步骤6
- 选择2：开始新任务，删除或保留旧任务（提示确认）
- 超时：默认选择2

---

### D2 - 多主skill匹配

**触发条件**：`calculate_match_score` 返回多个 ≥ 0.4 的候选skill

**提示格式**：
```
【技能选择】

根据您的需求，找到{count}个匹配的主skill:

1. {skill_name_1}
  说明: {desc_1}
  匹配度: {score_1:.1%}
  能力: {core_ability_1}
  路径: {path_1}

2. {skill_name_2}
  说明: {desc_2}
  匹配度: {score_2:.1%}
  ...

3. {skill_name_3}
  说明: {desc_3}
  匹配度: {score_3:.1%}
  ...

请选择 [1-{count}]: _
```

**显示规则**：
- 仅显示Top 3（即使更多）
- 匹配度百分比保留1位小数
- 太长描述截断到100字符

**处理逻辑**：
- 选择对应编号：使用该skill进入步骤4
- 超时：选择分数最高的skill

---

### D3 - 原子skill错误处理

**触发条件**：原子skill执行失败且影响评估 ≥ 5分

**影响评估示例**：
- 严重影响（≥5分）：后续有多个依赖，核心功能，数据一致性要求高
- 中等影响（3-4分）：部分依赖，非核心功能
- 轻微影响（≤2分）：无依赖，可恢复

**提示格式**：
```
【错误处理】

原子skill执行失败:
- 技能名称: {skill_name}
- 错误信息: {error_message}
- 重试次数: {retry_count}/3
- 影响评估: {影响等级}
- 影响说明: {影响说明}

后续任务中的 {dependent_skill_count} 个技能依赖于此技能:
- {dependent_skill_1}
- {dependent_skill_2}
...

处理方式:
1. 重试当前技能（建议）
2. 跳过当前技能，继续后续（风险: {risk_description}）
3. 取消整个任务
4. 请求人工介入修复（描述问题供用户调整后重试）

输入 [1/2/3/4]: _
```

**风险提示**：
- 跳过核心技能："可能导致最终结果不完整"
- 跳过数据修改："可能导致数据不一致"
- 跳过依赖项："后续技能可能失败"

**处理逻辑**：
- 1：返回步骤5，重试当前skill
- 2：标记skill为"跳过"，更新依赖，继续执行
- 3：进入步骤8输出失败结果，归档
- 4：记录详细错误供用户查看，暂停任务

---

### D4 - 执行计划调整确认

**触发条件**：恢复执行时检测到原计划不合理，需调整

**提示格式**：
```
【计划调整】

Reasoner检测到当前执行计划存在以下问题:

问题1: {issue_1}
问题2: {issue_2}
问题3: {issue_3}

建议调整方案:
- 原子skill顺序: 从 {old_order} 调整至 {new_order}
- 补充技能: {additional_skills}
- 删除冗余: {skipped_skills}

是否接受此调整并继续执行?
输入 [Y/n]: _
```

**处理逻辑**：
- Y/回车：接受调整，应用新计划，继续执行
- n：拒绝调整，返回步骤6，使用原计划
- 超时：默认接受调整

---

### D5 - 反思失败处理

**触发条件**：反思重规划达到3次上限仍未达标

**提示格式**：
```
【反思上限】

已进行 3 次反思重规划，仍未达到验收标准:

最终偏差: {deviation:.2%}
失败根本原因: {root_cause}
已调整: {adjustments_made}

可用选择:
1. 继续尝试（突破上限，需授权）
   - 预计成功率: {estimated_success_rate}
   - 额外耗时: ~{estimated_time}
2. 放弃并输出当前最佳结果
   - 任务状态: 部分完成
   - 保留已验证有效的产出
3. 暂停任务，手动调整后继续
   - 生成调整建议供您查看
   - 您可修改任务plan后保存

输入 [1/2/3] 或直接回车(推荐2): _
```

**处理逻辑**：
- 1：重置反思计数，继续步骤7（需用户明确授权）
- 2/回车：进入步骤8输出部分结果
- 3：暂停任务，生成建议文档，等待用户手动修改 `task_skill.md`

---

### D6 - 可固化文档提示

**触发条件**：任务完成，检测到结果可复用

**可固化判定条件**：
- 任务成功率 ≥ 80%（历史数据）
- 任务模式出现 ≥ 2次
- 不包含敏感信息（API密钥、密码等）
- 输出文档质量高（完整、清晰）

**提示格式**：
```
【任务完成】

任务ID: {task_id}
状态: {成功/部分成功}
执行时间: {duration}
产出数量: {output_count}

检测到此任务可能有可固化的文档:

建议文档:
1. {doc_title_1}
   - 类型: {doc_type}
   - 大小: {doc_size}
   - 用途: {doc_usage}
   - 路径: {doc_path}

是否移至项目docs/目录？

操作:
1. 是，复制到docs/并标注来源（自动）
2. 否，仅记录到索引
3. 稍后手动处理

输入 [1/2/3]: _
```

**自动复制逻辑**（选择1）：
```
目标路径 = "docs/tasks/{YYYY-MM}/{task_id}_{doc_name}"
复制文件: source -> target
添加注释到文档头:
"""
源任务: {task_id}
创建时间: {timestamp}
原始用户意图: {intent}
"""
更新docs/index.md索引
```

**处理逻辑**：
- 1：自动复制并索引
- 2：仅记录到月度索引，不移动文件
- 3：记录待办，等待用户手动操作
- 超时：默认2（仅索引）

## 3. 默认行为配置

### 3.1 超时策略

```python
DEFAULT_TIMEOUT_BEHAVIORS = {
    "D1": "开始新任务",      # 5分钟超时
    "D2": "选择最高分",      # 30秒超时
    "D3": "重试",           # 3分钟超时
    "D4": "接受调整",        # 2分钟超时
    "D5": "输出当前结果",    # 5分钟超时
    "D6": "仅索引",         # 1分钟超时
}
```

### 3.2 无用户输入处理

当用户未提供明确输入时的 fallback：
```
if timeout:
    执行默认行为
elif 输入为空:
    使用默认行为
elif 输入无效:
    重新提示（最多3次），3次后使用默认
```

## 4. 提示显示优化

### 4.1 信息分级

```
优先级1（必显）:
- 任务ID、技能名称、错误信息
- 选项列表、输入提示

优先级2（重要）:
- 已完成数量、重试次数、影响评估
- 偏差值、原因分析

优先级3（补充）:
- 详细描述、示例、次要信息
- 环境信息、时间戳

显示限制:
- 总行数 ≤ 50行
- 关键决策信息在前30行
- 可折叠详细信息（用...表示）
```

### 4.2 历史上下文摘要

当任务恢复时，提供简洁上下文：
```
【任务恢复上下文】
上次执行到: 原子skill_X (已完成)
当前进度: 3/8 (37.5%)
最近异常: skill_Y 第2次重试失败
剩余估计: ~5分钟
建议: 继续执行
```

## 5. 特殊场景处理

### 5.1 用户不在时（无响应）

**超时后自动行为**：
1. D1（任务发现）：启动新任务
2. D2（技能选择）：选择最高匹配度
3. D3（错误处理）：重试当前skill（≤3次）
4. D4（计划调整）：接受调整建议
5. D5（反思失败）：输出当前最佳结果
6. D6（文档固化）：仅记录索引

**自动模式配置**：
```python
AUTO_MODE_CONFIG = {
    "max_auto_retries": 3,
    "auto_accept_adjustments": True,
    "auto_output_on_failure": True,
    "skip_user_confirm_on_critical": False,
}
```

### 5.2 用户频繁介入

**检测**：用户在单个任务中做出选择次数 > 5

**处理**：
- 提示用户："是否需要切换到自动模式？"
- 提供配置选项减少决策点
