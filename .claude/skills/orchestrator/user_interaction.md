## D1 - 未完成任务发现

**触发条件**：在 `tasks/current/` 下找到相关未完成任务

```
【任务发现】

检测到相关未完成任务:
- 任务ID: {task_id}
- 用户意图: {intent_summary}
- 已完成: {completed}/{total} 个原子skill
- 创建时间: {create_time}

请选择:
1. 继续执行未完成任务（从断点恢复）
2. 开始新任务（忽略旧任务）

输入 [1/2] 或直接回车(默认2): _
```

**处理逻辑**：
- 选择1：继续执行（跳至步骤6）
- 选择2：删除或保留旧任务（提示确认），开始新任务
- 超时：默认选择2（开始新任务）

---

## D2 - 多主skill匹配

**触发条件**：`calculate_match_score` 返回 ≥ 2 个 ≥ 0.4 的候选skill

```
【技能选择】

根据您的需求，找到 {count} 个匹配的主skill:

1. {skill_name_1}
   说明: {desc_1}
   匹配度: {score_1:.0%}
   能力: {core_ability_1}

2. {skill_name_2}
   说明: {desc_2}
   匹配度: {score_2:.0%}
   ...

3. {skill_name_3}
   说明: {desc_3}
   匹配度: {score_3:.0%}
   ...

请选择 [1-{count}]（输入数字）: _
```

**显示规则**：
- 仅显示Top3（即使匹配更多）
- 匹配度保留整数百分比
- 描述截断到80字符

**处理逻辑**：
- 输入对应编号：使用该skill进入步骤4
- 超时：选择分数最高的skill

**多skill组合检测**：
当复杂度 ≥ 7 时，自动显示组合建议：
```
💡 检测到这是一个复杂任务，可能需要多skill协同：

推荐组合: bug-solver + code-optimizer
- bug-solver: 修复问题
- code-optimizer: 优化重构

是否使用此组合? [Y/n]: _
```

## D3 - 原子skill错误处理

**触发条件**：原子skill执行失败，且影响评估 score ≥ 5（严重影响）

```
【错误处理】

原子skill执行失败:
- 技能名称: {skill_name}
- 错误信息: {error_message[:200]}
- 重试次数: {retry_count}/3
- 影响评估: {影响等级} (score={score})

后续有 {dep_count} 个技能依赖于此:
- {dep_skill_1}
- {dep_skill_2}
...

处理方式:
1. 重试当前技能（推荐）— 重新执行，最多3次
2. 跳过当前技能，继续后续 — 风险：{risk_desc}
3. 取消整个任务 — 归档当前状态，输出结果
4. 请求人工介入 — 暂停任务，等待您修复后继续

输入 [1/2/3/4]（默认1）: _
```

**风险提示**：
- 跳过核心技能："可能导致最终结果不完整"
- 跳过数据修改："可能导致数据不一致"
- 跳过依赖项："后续技能可能失败"

**影响评分参考**：
| score | 等级 | 默认行为 |
|-------|------|---------|
| ≥ 5 | 严重 | 暂停，等待选择 |
| 3-4 | 中等 | 提示用户，询问选择 |
| ≤ 2 | 轻微 | 记录日志，自动继续 |

## D4 - 执行计划调整确认

**触发条件**：任务恢复时检测到原计划不合理

```
【计划调整】

Reasoner 检测到当前执行计划存在以下问题:

问题1: {issue_1}（{impact_1}）
问题2: {issue_2}（{impact_2}）

建议调整方案:
- 原子skill顺序: 从 [{old_order}] 调整至 [{new_order}]
- 补充技能: {additional_skills}（{count}个）
- 删除冗余: {skipped_skills}（{count}个）

⚠️ 此调整不会改变主skill和最终目标

是否接受此调整并继续执行?
输入 [Y/n]（默认Y）: _
```

**调整限制说明**：
- ✅ 允许：微调原子skill顺序、补充同类型原子skill
- ❌ 禁止：替换主skill、新增未注册技能

**处理逻辑**：
- Y/回车：接受调整，应用新计划，继续执行
- n：拒绝调整，返回原计划继续执行
- 超时：默认接受调整

## D5 - 反思失败处理

**触发条件**：反思重规划达到3次上限，仍未达标

```
【反思上限】

已进行 3 次反思重规划，仍未达到验收标准:

最终偏差: {deviation:.1%}
失败根本原因: {root_cause}
已尝试调整: {adjustments_made}

可用选择:
1. 继续尝试（突破上限，需您明确授权）
   - 预计成功率: {estimated_success_rate}
   - 额外耗时: ~{estimated_time}
2. 放弃并输出当前最佳结果（推荐）
   - 任务状态: 部分完成
   - 保留已验证有效的产出
3. 暂停任务，手动调整后继续
   - 生成调整建议供您查看
   - 您可修改 task_skill.md 后保存

输入 [1/2/3] 或直接回车(推荐2): _
```

**处理逻辑**：
- 1：重置反思计数，继续步骤7（需用户明确授权）
- 2/回车：进入步骤8输出部分结果
- 3：暂停任务，生成建议文档，等待用户手动修改

## D6 - 可固化文档提示

**触发条件**：任务完成，检测到结果可复用

**可固化判定标准**：
- 任务成功率 ≥ 80%
- 任务模式出现 ≥ 2次
- 不包含敏感信息（API密钥、密码等）
- 输出文档质量高（完整、清晰）

```
【任务完成】

任务ID: {task_id}
状态: {成功/部分完成}
执行时间: {duration}
产出数量: {output_count}

检测到可固化的文档:
1. {doc_title}
   - 类型: {doc_type}
   - 大小: {doc_size}
   - 用途: {doc_usage}

是否移至项目 docs/ 目录？

操作:
1. 是，复制到 docs/ 并标注来源（自动）
2. 否，仅记录到月度索引
3. 稍后手动处理

输入 [1/2/3]（默认2）: _
```

**自动复制逻辑**（选择1）：
```
目标路径 = "docs/tasks/{YYYY-MM}/{task_id}_{doc_name}"
复制文件 → 添加来源注释 → 更新 docs/index.md
```
